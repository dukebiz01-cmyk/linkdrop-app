## 절대 원칙

- **§0:** 드로피는 "공유"가 아니라 **친구가 쿠폰 사용/주문/예약 완료(전환)**에. 문구 이 프레임 유지.

- **studio-build는 미러 파일** — 아래 [목적 수신]이 복잡/위험하면 콘텐츠·라우팅만 하고 studio-build 변경은 **STOP·보고.**

## 5항목 (순서 · 라우팅 · 가이드 콘텐츠)

### ① 영상·콘텐츠로 카드를 만들래요 → `/studio-build?purpose=정보`
- **만드는 법:** 유튜브·인스타 영상 링크 한 줄만 붙이면, 링고AI가 영상 속 핵심을 카드로 정리해줘요.
- **좋은 점:** 긴 영상 안 봐도 핵심이 카드 하나에. 카톡으로 바로 공유돼요.
- **공개하면:** 탐색에 올라가 다른 사람들이 보고 공유해요. 카드가 널리 퍼집니다.
- CTA: **이 목적으로 시작하기 →**

### ② 우리 가게를 알리고 싶어요 → `/studio-build?purpose=쿠폰` ⭐추천(상인)
- **만드는 법:** 가게 등록 후 쿠폰·예약 카드를 만들어요. 링고AI가 문구·조건을 도와줍니다.
- **좋은 점:** 손님이 카톡에서 바로 쿠폰 받고 예약. 카드가 살아있는 알림판이 돼요.
- **공개하면:** 탐색 노출 → 손님·친구가 공유 → 받은 사람이 **쿠폰을 쓰거나 예약을 완료하면**, 만든 사람과 전달한 사람이 **드로피**를 받아요.
- CTA: **이 목적으로 시작하기 →**

### ③ 상품을 팔고 싶어요 → `/studio-build?purpose=구매`
- **만드는 법:** 제철 식재료 같은 상품을 주문 카드로. 링고AI가 상품·가격을 정리해줍니다.
- **좋은 점:** 영상 보고 "사고 싶다"를 그 자리에서 주문으로. 카톡에서 바로 구매돼요.
- **공개하면:** 탐색 노출 → 공유가 퍼지고 → 친구가 **실제로 주문하면**, 만든 사람과 전달한 사람에게 **드로피**가 분배돼요.
- CTA: **이 목적으로 시작하기 →**

### ④ 받은 혜택을 관리할래요 → `/me` (내 지갑) 🆕
- 받은 **쿠폰** · 충전한 **캐시** · 모은 **드로피**가 한곳에.
- 내가 공유한 카드를 친구가 **쓰면(전환)** 드로피가 쌓여요.
- CTA: **내 지갑 보기 →**

### ⑤ 먼저 둘러볼래요 → `/explore`
- 다른 사람들이 만든 카드를 구경하고, 마음에 드는 카드를 카톡으로 공유해보세요. 친구가 **쓰면(전환)** 나도 **드로피**를 받을 수 있어요.
- CTA: **둘러보기 →**

## AI 가이드 UI
- 각 목적 = **인라인 아코디언**(시트/포털 아님, #418 안전). 탭 → 그 아래 펼침: **만드는 법 / 좋은 점 / 공개하면** 3블록 + 하단 CTA.
- 한 번에 하나만 펼침(다른 목적 접힘) 권장.
- ④·⑤는 짧으니 1~2줄 + CTA.

## 화살표 (UI 정리)
- **목적 행 우측 = 셰브론(∨)**, 펼침 시 회전(rotate-180). "여기 눌러 펼침" 방향.
- **가이드 하단 CTA = 오른쪽 화살표(→, ArrowRight):** "이 목적으로 시작하기 →"(①②③) / "내 지갑 보기 →"(④) / "둘러보기 →"(⑤).

## 라우팅 (RoleHome 배선)
- ① → `navigate({to:"/studio-build", search:{purpose:"정보"}})`
- ② → `.../studio-build?purpose=쿠폰`
- ③ → `.../studio-build?purpose=구매`
- ④ → `/me` (지갑 탭이 있으면 해당 탭)
- ⑤ → `/explore`
- **create-wizard 참조 전부 삭제.** onCreate/onExplore 대신 목적별 핸들러 or purpose 인자 전달.

## studio-build 목적 수신 (신설 — 미러 주의)
- Route에 `validateSearch` 추가: `{ purpose?: "정보"|"쿠폰"|"예약"|"구매" }`.
- **초기화 시 purpose param 있으면 매칭 팔레트 블록 자동 켜기:**
  - `쿠폰` → 쿠폰 연결 블록 / `예약` → 예약 캘린더 블록 / `구매` → 상품 블록 / `정보` → 기본(영상, 추가 블록 없음).
  - **정확한 블록 ID는 studio-build 팔레트(`category:"purpose"`)에서 확인해 매핑.**
- **미러 안전:** CardBody(스튜디오↔손님 거울)에 영향 없이 **초기 블록 상태만 프리셋.** 확인 후 진행. 애매하면 STOP.
- purpose 없이 진입(bottom-nav 등)은 **기존대로 무영향**(하위호환).
