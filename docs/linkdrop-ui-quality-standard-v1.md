# LinkDrop UI 품질 정본 v1 (2026-07-16, Duke 승인 대기 → 승인 시 락)

목적: 화면마다 제각각인 품질을 한 기준으로 통일해 재작업을 없앤다.
적용 범위: 링고 셸(홈·스튜디오) 및 이후 만드는 모든 표면. 기존 화면은 어긋난 곳을 "발견 목록"으로 보고만 (수술은 승인 후).

## 1. 원칙 5개

1. **60대 가독 최우선** — 본문 13.5px 미만 금지, 캡션도 11px 미만 금지. 대비 흐린 회색-위-회색 금지.
2. **카피 대신 상태** — "듣고 있어요" 같은 설명 문구보다 ON/OFF·색·모션으로 상태를 말한다. 문구는 질문·안내 1문장만.
3. **한 화면 한 주인공** — 화면당 목적색 채움(주 버튼/강조)은 1개 계열만. 나머지는 보더·회색 위계.
4. **부유·돌출 0** — 모든 요소는 자기 컨테이너 라운드 안에 수납. absolute 돌출·negative margin 금지.
5. **가짜 숫자 0** — 데모·예시는 "예시" 라벨 명시. 로딩/빈/에러 3상태를 모든 데이터 표면에 설계.

## 2. 토큰 (전 표면 공통)

- **색**: 잉크 #1A1A2E · 목적색 3종(정보 #1D4ED8 / 예약 #7F1D3A / 상품 #0F766E) · 서피스(배경 #F7F6F2, 카드 #FFFFFF) · 보더 #E3E1DA(기본)·#D8D6CE(강조) · 보조텍스트 #6B6961 · 캡션 #9A988F
- **목적색 규칙**: 현재 카드 목적이 화면의 액센트를 결정 — 스텝퍼 현재 번호·덱 배지·마이크 레일·스피커 ON·주 버튼이 전부 같은 목적색.
- **라운드**: 카드 14px · 패널 18px · 알약/스위치 999px. 한 화면에 이 3종 외 금지.
- **간격**: 8pt 그리드 (8/12/16/24). 블록 사이 최소 12px — 그라데이션·그림자가 이웃 블록 침범 금지(컨테이너 내부 클립).
- **타이포**: 화면 제목 16/700 · 섹션 제목 13/700 · 본문 13.5~14/400 · 캡션 11~12. 폰트 시스템 산세리프 단일.
- **터치**: 모든 탭 대상 44px 이상 (시각 크기는 작아도 히트 영역 확보).
- **그림자**: 떠 있는 것(캡슐·패널)에만 1단계. 정적 카드는 보더로만.

## 3. 컴포넌트 규칙

- **버튼 위계**: 주(목적색 채움, 화면당 1계열) > 보조(흰 배경+보더) > 텍스트 링크. 비활성 = #E3E1DA 배경 + #9A988F 글자.
- **칩**: 999px 알약, 보더형, 13px, 한 줄 최대 2~3개. 행동 칩은 현재 단계 맞춤만.
- **배지(번호)**: 26px 원. 3상태 = 완료(목적색 채움+✓) / 현재(목적색 2px 테두리+펄스) / 대기(회색 보더+번호). 전 표면 동일 규격.
- **스위치 가족**: 마이크 레일(ON/OFF 각인·손잡이 52px·목적색 채움)과 스피커 토글(34px 원·ON=목적색 12~15% 배경+목적색 아이콘)은 같은 문법 — "소리 계열 = 목적색 스위치".
- **아이콘**: 프로젝트 기존 아이콘셋(lucide 계열) 단일 사용, 스트로크 굵기 통일. 이모지 UI 사용 금지 (콘텐츠 텍스트 제외).
- **입력**: 12px 라운드, 배경 #FAFAF7 + 보더, placeholder는 캡션 색.

## 4. 모션 규칙

- 의미 있는 모션만 3종: 펄스(지금 차례) · 파형(듣는 중) · ✓전환(완료 700ms). 장식용 반복 애니메이션 금지.
- 전환 120~250ms ease. 레이아웃을 밀어내는 애니메이션 금지 (opacity/transform만).
- 심각한 상태(에러)는 모션 없이 정적 표시.

## 5. 상태 3종 의무

모든 데이터 표면(성과 스트립·1층 그리드·피드 타일)은 로딩(스켈레톤 또는 은은한 자리) / 빈("아직 쌓이기 전" 계열 정직 문구 + 다음 행동 1개) / 에러(재시도 1버튼) 를 갖춘다. 숫자 0 하드코딩으로 빈 상태를 때우는 것 금지.

## 6. 적용 절차

1. 이번 통합 슬라이스(작업 2·5·8·9·10·11)는 이 정본을 기준으로 구현.
2. 작업 중 기존 화면에서 정본 위반을 발견하면 — 고치지 말고 [발견 목록]으로 최종 보고에 첨부 (파일:위치·위반 조항). 수술은 Duke 승인 후 별도 슬라이스.
3. 이 문서와 코드가 충돌하면 이 문서가 정본. 문서 개정은 Duke 승인으로만.
